Day 2 · Token与概率：语言模型的底层逻辑

它为什么有时候「胡说八道」？

📅 2026-04-09 Wed 📦 第一模块 · 地基篇 ⏱️ 预计 40 分钟
🔥 开场

昨天我们留了一个问题：如果模型每一步都选概率最高的那个词，答案会不会变得很无聊、很重复？

答案是——会，而且会严重到不可用。

但现实中，大模型的输出经常让你惊艳。这种「创造力」的假象，背后是一套精妙的概率采样机制。 今天你将看到，「随机性」不是 bug，而是大模型最重要的设计特性之一。

10 min

Why — 为什么必须理解 Token 与概率？

先给你三个真实场景：

场景一： 你让 Kimi 帮你写一份渠道策略报告。生成了两个版本，一个中规中矩，一个有亮眼的洞察但某些细节不太对。你选哪个？——这背后是 Temperature 参数在起作用。

场景二： 你问 ChatGPT 同一个问题三次，每次答案都不一样。这是模型不靠谱吗？——不，这是概率采样的正常表现。

场景三： 你让 AI 给广告文案起名字，要求「有创意」。结果它要么太安全，要么太离谱。——这是采样策略没选对。

🔑 关键认知

如果你不理解 Token 和概率，你就永远只能被动接受模型的输出，而不是主动驾驭它。

20 min

What — Token 与概率的底层机制

第一层：Token——语言的「最小可操作单元」

Tokenizer：模型的「翻译官」

在你的文字到达模型之前，有一个被大多数人忽略的角色——Tokenizer（分词器）。

你输入：「腾讯广告2026年渠道策略」 Tokenizer 切分：[腾讯] [广告] [20] [26] [年] [渠道] [策略] → 7 个 Token 英文输入：「The quarterly performance report」 切分：[The] [ quart] [erly] [ performance] [ report] → 5 个 Token

注意几个反直觉的事实：

  • 切分方式不是固定的。不同模型的 Tokenizer 不同。
  • 生僻词会被切得更碎。模型处理越常见的词越「轻松」。
  • Token 数 = 成本 = 限制。API 按 Token 收费；上下文窗口按 Token 计算。
📊 类比

Tokenizer 就像你管理渠道时的数据指标体系。你需要把复杂的现实切分成合适粒度的指标——毛利率、回款周期、客户增长率——这些就是你的「Token」。粒度选得好，管理才高效。

第二层：概率分布——模型的「世界观」

从 Token 到向量：嵌入（Embedding）

每个 Token 被转换成向量——一长串数字，在超高维空间中的坐标。语义相近的词，距离就近。

Logits → Softmax → 概率

模型输出原始分数（Logits），通过 Softmax 转换成概率：

Logits: { 不错: 5.2, 很好: 4.8, 晴朗: 4.1, 巧克力: -12.3, ... } ↓ Softmax 概率: { 不错: 0.18, 很好: 0.12, 晴朗: 0.06, 巧克力: 0.0000004, ... }

第三层：采样——「选谁」的学问

Greedy Decoding（贪心解码）

每次都选概率最高的 Token。结果：永远是最「安全」、最「平庸」的答案。

Temperature Sampling（温度采样）

Temperature 控制概率分布的「陡峭程度」：

Temperature效果类比
= 0等同于 Greedy，完全确定性严格执行标准流程
0.3-0.7偶尔也能选到概率稍低的有限度的灵活空间
= 1.0按原始概率分布采样自由发挥
> 1.0概率被「拍平」，更随机天马行空，风险极高

Top-K 与 Top-P

Top-K：只从前 K 个候选中选。Top-P：累积概率达到 P 才停。Top-P 是自适应的——模型确信时收窄，犹豫时放宽。

📊 类比

Top-K 是固定名额制——「只考核 Top 10 代理商」；Top-P 是动态达标制——「达到业绩总额 90% 的代理商都纳入」。

完整的生成过程

[你的输入] → [Tokenizer] → [Embedding] → [Transformer处理]
→ [输出 Logits] → [Temperature 调整] → [Softmax]
→ [Top-P 筛选] → [采样] → 输出一个 Token
→ 加入输入序列，重复…
8 min

So What — 这改变了什么？

1. 你获得了一个新的诊断框架

症状可能原因调整方向
答案太保守Temperature 太低提高 Temperature
答案太发散Temperature 太高降低 Temperature / 收紧 Top-P
中文不流畅Tokenizer 问题换中文优化更好的模型
长文后半段质量下降自回归路径依赖分段生成或先列大纲
每次答案都不一样概率采样随机性Temperature 调低或设 seed

2. 对广告行业的直接启示

腾讯广告的智能投放系统，底层逻辑和大模型生成文字有惊人的相似性：

  • 广告推荐 = 从候选广告池中「采样」一条广告
  • CTR 预估模型 = 给每条广告一个「概率分布」
  • Explore vs Exploit = 广告系统的「Temperature」

3. Token 经济学

2026 年的 AI 应用，成本核心就是 Token。输入 Token 多 → 理解更好但成本高；Tokenizer 效率直接影响性价比。

✅ 今日 Checkpoint

为什么大模型每次回答同一个问题，答案可能都不一样？这是模型不靠谱，还是一种设计？

（参考：这是设计。每个 Token 都是从概率分布中采样的，只要 Temperature > 0，就存在随机性。不同任务需要不同程度的随机性——写代码要确定性，写创意要随机性。）

🎯 明日悬念

模型的这些「概率判断」是从哪来的？

答案是：训练。 但训练不止一种。明天我们讲 Pre-train vs Post-train——同一个架构，训练方式不同，出来的「性格」天差地别。

📌 知识卡片

1Tokenizer 把文字切成 Token
不同模型切法不同。Token 数 = 成本 = 性能瓶颈。
2概率分布是模型的「意见」
Embedding → Transformer → Logits → Softmax → 概率。
3Temperature 控制随机性
低 = 保守确定，高 = 多样冒险。T=0 等于 Greedy。
4Top-P 是自适应采样
模型确信时收窄，犹豫时放宽。比 Top-K 更智能。
5随机性不是 bug，是特性
大模型的「创造力」来自于不总是选最高概率——这是刻意的设计。
